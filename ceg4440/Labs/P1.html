Get started with Android Development


The purpose of this lab/project is
  • To make you aware that an Android device is a fully networked computer system
  • To get you started with software design and development on the Android platform. This first project focuses on a few simple tasks.
IDE == Android Studio with IntelliJ Idea (recommended), or Eclipse; Report == a single PDF file with descriptions as suggested. Refer to the Android developer's site http://developer.android.com/ for detailed guides. Generally speaking, build the APKs for Android 5.1. All lab work is expected to be performed on Linux systems connected to Android devices (phones or tablets).

Task 1: Android Networking APKs

Download from Google Play and install
  • An SFTP/SSH server
  • An ssh client/terminal
  • A local terminal on your Android device.
Some example APKs are web.oss.sshsftp, sk.vx.connectbot, com.spartacusrex.spartacuside. But, you are welcome to choose other APKs that fit the description.
Deliverables:
  • Please include in your Report your experience of using these apps.
  • Include several screenshots running the APKs in the emulator or real device in your write-up.

Task 2: Android Developement

Use IDE to build "Hello-This-Is-Me" APK and run it within Android Emulator or on a device of your choice. It is highly recommended that you first build the "Hello World" following the developer.android.com tutorial.
  1. Display text "1. CEG 4440 Android Security -"+ <your name> on the screen by using code.
  2. Display text "2. CEG 4440 Android Security -" + <your name> on the screen by XML
  3. Display your photo on the screen.
  4. Use two different layouts in Android to display the text "CEG 4440 Android Security - " + <your name> followed by your photo.
  5. Combine all the above into one APK with multiple screens with transitions in the order shown.
Deliverables:
  • Please include in your Report your experience on the installation and verification of Android Studio.
  • Include several screenshots running the APK in the emulator (or real device) in your write-up.
  • Your project files (java, xml, etc) exported from your Android project as an archive.
  • HelloThisIsMe.apk
