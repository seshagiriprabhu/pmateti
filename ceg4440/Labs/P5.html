Code Injections and Mitigations


The purpose of this lab is to get you deeper into code injection attacks in PCs and Android devices.

Task 1: Bash Shellshock Injection (CVE-2014-6271)

Task: (i) Visit a CVE database, and read all about CVE-2014-6271, and related CVEs, e.g., CVE-2014-7187. (ii) Download and install (latest) bash (ARM) binary into your Android device. (iii) Verify if your laptop Linux bash (already installed) has this vulnerability. (iv) Verify if your Android device bash has this vulnerability.

Deliverables:
  • Short (one or two paragraph) reports on each of the above items.

Task 2: Make modret.c Skip Lines of Code

Goal: Understand the stack smashing buffer exploit /thoroughly/. You are permitted to do this task on a Linux PC.
  1. Improve the code of =exploit4.c= of AlephOne so that there are no warning messages whatsoever from =gcc= even after using the flags as in =gcc -ansi -pedantic -Wall=
  2. Reduce the size of their compiled binaries by at least 5% as seen by the =size= command under the text column. Make sure no functionality is lost. Removing printf's amounts to changing functionality. Optimized rewriting of portions of source code is permitted. You must use the same gcc (optimization or other) flags.
  3. Compile and run =modret.c=. (Instructions to compile and examine the assembly code are embedded as comments in the code.) Your goal is to make this process print
    • "now, x == 66"
    • "now, x == 44"
    • "now, x == 11
    by invoking the modret program three times with chosen integers. /You may need to modify the numbers appearing with the modret procedure./ Record your process of discovering these numbers. A relevant code segment from this file is reproduced here: 
    x = 0x11;
modret(y, z, 0x234); /* easy to recognize number */
x += 0x22;
x += 0x33;
printf("now, x == %x\n", x);

Deliverables:

  • =exploit4-improved.c=
  • =size exploit4-improved=
  • =modret= input + output showing three values of x.

Task 3: StageFright

Task: There are (Oct 2015) a few APKs on Google Play that (claim to) detect if the StageFright library within your device is vulnerable still or patched up. Install one such APK and report on its results.
Deliverables:

  • screenshot of the installed APK
  • screenshot of the results
  • short summary and critique of the APK

Task 4: ASLR in Android

Task: Develop a program/APK that we can install within Android that tells us if ASLR is enabled or not, and offers evidence of the conclusion. Implement this in C/C++/ Java/ Scala/ Python. It need not be an APK. It is acceptable if it can be invoked via =adb shell=. Suggestion: Try this program out first on a Linux PC and then cross-compile it for Android.

Warning: Conceptual discussions on this task among students/faculty is permitted, but not implementation details, certainly not code sharing. Code must be /yours/ (not "borrowed" from the web or whereever).
Deliverables:

  • Source code of program/APK. Well commented
  • Its design description.
  • Example outputs of test runs.
