Development of Networked Android Apps


The purpose of this lab/project is to get you deeper into Android development which is a prerequisite to security discussion. Build for Android Lollipop.

Task 1: Networking Android Virtual Devices

Refresh on networking essentials: ../Lectures/Refreshers/index.html. Understand the section on Emulator Networking in http://developer.android.com/ tools/ devices/ emulator.html . Task: Network three devices so that an ssh client-server setup is demonstrable between any pair. These devices can be
  • AVDs on the PC(s)
  • Real devices
The AVDs should differ slightly from each other so we can id them easily. We will use this network in Task 3.
Deliverables:
  • Routing tables.
  • Proof that the network works.

Task 2: Shrink OpenSudoku Given its Source Code

Read our overview of Open Sudoku in ../Lectures/AppDevelop. Also, visit http://opensudoku. moire. org/, which has a source download link. Google Play Store has OpenSudoku as a free app. (It is highly recommended that you install this on your real device.) Install this on your AVD using adb. It is helpful, but not essential, to learn to solve Sudoku puzzles (using OpenSudoku.apk).
  1. Task: Build as-is from the source and run this APK. Write a paragraph or two on how you built it. Link to source is in the overview of Open Sudoku. Capture 4+ interesting screenshots. Especially capture the screens that will go away -- see below.
  2. Task: The app as-given has a range of functionality. Chop off all this functionality, and bring it down to the essentials of just interactively solving Sudoku. E.g., it can download Sudoku examples from various places. Make sure to ACK the OpenSudoku project on the first screen. Change to edu.wright.ceg4440. Build and run this stripped down APK. Capture 4+ interesting screenshots.
We will use the stripped down OpenSudoku in Task 3. Also, this is an efficient way to become familiar with howto develop a "medium-size" APK, without actually writing all of it. Deliverables
  • Screenshots
  • Two .apk files -- before and after
  • Your project files, after the strip down, as an archive exported from IDE.

Task 3: Collaborative Sudoku

Modify the app so that different players can collaborate over a network. Task: Add a network based mechanism that synchronizes the display of the current puzzle being solved. Ignore authentication issues. Ignore complicated synchronization issues. You may use one of two ways of networking the Collaborative Sudoku.
  1. Use the network established in Task 1 to connect three AVDs or real devices, each running the modified OpenSudoku. Unless you already have a good understanding of TCP/IP, this can be hard.
  2. Use https://developers.google.com/cloud-messaging/ .
Deliverables:
  • Pseudo code level design description
  • Explicit list of source code changes made
  • apk
