1.1 Build Issues

  1. The 2012v can be imported into Eclipse without any hassle. Use this if you prefer.
  2. The 2014 version is by a new author and uses IntelliJ Idea and even then it has a few Gradle dependencies that need to be resolved. A solution to 2014v build issues was posted on Piazza, summarized below.
    1. edit the build.gradle file. Change the classpath from com.android.tools.build:gradle:0.8.+ to com.android.tools.build:gradle:0.9.+.
    2. Change com.android.support:appcompat-v7:+ to com.android.support:appcompat-v7:19.0.+.
    3. To get rid of the of the theme.cap error you have to import the android-support-v13.jar.

2 Collaborative Sudoku

  1. Sudoku Server
    1. The "first" Sudoku invoked.
    2. Accepts clients who will "input" entries.
    3. Sends the current display of the puzzle to the clients.
  2. Sudoku Clients
    1. Connects to a Sudoku server
    2. Receives the current display of the puzzle.
    3. The inputs by the user cause puzzle display updates as usual
    4. But also sends the inputs to the server
  3. Required Reading http://developer.android.com/training/basics/network-ops/index.html
  4. AVD Networking
  5. Google Cloud Messaging Service https://developers.google.com/cloud-messaging/
