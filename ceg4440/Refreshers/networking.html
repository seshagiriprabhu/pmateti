1 Networking Layers

1.1 Layered Models

Figure 1: OSI 7 Layers v DoD 4 Layers (numbered upwards) [Source: Web]

1.2 Layered Models #2

  1. TCP/IP Suite is a collection of protocols.
  2. Ethernet, etc: Layer 1 (lowest) of the 4 layer DoD Network Protocol Stack Model.
  3. IP is in Layer 2. That is, it is not concerned with hardware details.
  4. TCP and UDP are in Layer 3.
  5. Application protocols (Layer 4): http; ssh, …
  6. DoD = US Department of Defense

2 Networking Hardware Technology

2.1 Ethernet

  1. Responsible for delivering data over the hardware media.
  2. Every Ethernet frame has two addresses: Sender's MAC address, Receiver's MAC address; 6 bytes + 6 bytes.
  3. Manufaturer sets the MAC address. Given a MAC address, we can discover the manufacturer.

3 Networking Protocols

3.1 IP

  1. Every IP datagram (packet) has Sender's IP address, Receiver's IP address
  2. Data payload is one of: TCP packet, UDP packet, …
  3. The Receiver's IP address makes it possible to route the datagram.

3.2 IP Addresses

  1. IPv4: 4 bytes + 4 bytes.
    1. Each byte value is written in decimal. Separated by a dot.
    2. Ex: 192.168.17.243; 127.0.0.1
  2. IPv6: 16 bytes + 16 bytes.
    1. Each 2-byte value written as 4 hex digits. Separated by a colon.
    2. Ex: 2001:0db8:85a3:08d3:1319:8a2e:0370:7348
    3. The world was supposed to have switched to IP6 in 2008. Real Soon Now.
  3. IP addresses are set at boot time by the OS.
    1. IP addresses are assigned by https://www.iana.org/.
    2. There are private ranges of IP addresses that we can choose without permission.
    3. Given a public IP address, we can discover its rough location.

3.3 IP Routers

  1. Route = a path from sender to receiver, via 0 or more (intermediaries known as) routers/ gateways
  2. Router == computer system specializing in routing
    1. Multiple network cards
    2. Routing Table
  3. ARP Address Resolution Protocol: Needs to know the IP address to Ethernet address mapping of the LANs

3.4 Port Numbers

  1. OS produced 16-bit (abstract) numbers; now "standardized" in /etc/services
  2. TCP port# separate from UDP port#
  3. The port#s are part of the TCP packet
  4. Privileged: 0 .. 1024; Ordinary users should use: > 1024
  5. http://www.iana.org/assignments/port-numbers
  6. Sockets v Ports

3.5 TCP Transmission Control Protocol

  1. Segments = TCP packets, contain
    1. sender's and receiver's port numbers
    2. data payload
  2. TCP is connection-oriented
    1. 3-way hand shake to establish a connection
    2. 4-way hand shake to terminate a connection
  3. TCP is reliable but slower :: UDP unreliable but faster
  4. TCP has many features: congestion adjustment, duplex, …
  5. Several variations in TCP implementations E.g., read Tahoe, Reno, New-Reno, SACK and Vegas 2005. Reference.
  6. ./TCPIP-CEG4420.ppt TCP/IP Refresher, 2014. Required Reading.

5 Network Security

5.1 Security Requirements

  1. We use the term message as an abstraction of a series of related packets that can be defined rigorously in the context of a network protocol.

5.2 Security Requirements

  1. Message integrity is the guarantee that the (i) body content, (ii) the identity of recipient, and (iii) the identity of the sender have not been altered (in transit).
  2. Delivery: We expect a message to be received by the intended in a finite (say a few seconds) amount of time after being sent.

5.3 Security Requirements

  1. Non-repudiation is often considered a legal concept. But, here we are concerned with (i) a receiver after having received a message denies receiving, (ii) a sender after having sent a message denies sending. We wish to have proof of delivery of a message.

5.4 Security Requirements

  1. A message may be intercepted, without violating the integrity. This is loss of confidentiality. It is often unstated but the message is intended for the eyes of the named recipient.
  2. Authenticity establishes to the recipient that the message was indeed composed by the sender.

5.5 Passive Exploits

  1. Passive: Do not alter the network stream
  2. Sniffing
  3. Traffic Analysis

5.6 Active Exploits

  1. Many of these discussed in detail later.
  2. Active: Alter the network stream
  3. IP address spoofing
  4. MitM (Man-in-the-Middle) Attacks
  5. DoS (Denial of service) attacks
  6. DDoS (Distributed Denial of service) attacks

5.7 Active Exploits #2

  1. Insertion Attacks
  2. ARP Poisoning in the LAN
  3. DNS Poisoning
  4. Sniper: Terminate a connection
  5. Hijacking a connection
